Introduction to HTML

HTML, which stands for HyperText Markup Language, is the foundational language used to create and design webpages. It provides the basic structure for web documents, allowing you to format text, create links, embed images, and organize content in a structured manner.

What is HTML?

HTML is a markup language that uses a system of tags to define elements within a document. These tags are enclosed in angle brackets and are used to format and display content on the web. HTML is essential for creating websites as it structures the content in a way that browsers can interpret and render.

Basic Structure of an HTML Document

An HTML document is structured with a series of elements. Hereâ€™s a basic example:


<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    <h1>Welcome to My Website</h1>
    <p>This is a paragraph of text.</p>
    <a href="https://www.example.com">Visit Example</a>
  </body>
</html>

HTML Tags and Elements

HTML uses tags to create elements. Tags come in pairs: an opening tag and a closing tag. Here are some common tags:

  • <h1> to <h6>: Headings of different levels, where <h1> is the largest and <h6> is the smallest.
  • <p>: Defines a paragraph.
  • <a>: Defines a hyperlink. The href attribute specifies the link destination.
  • <img>: Embeds an image. The src attribute specifies the image source, and alt provides alternative text.
  • <ul> and <ol>: Define unordered (bulleted) and ordered (numbered) lists, respectively. List items are defined with <li> tags.

Attributes

HTML elements can have attributes that provide additional information about the element. Attributes are placed in the opening tag and come in name-value pairs. For example:


<a href="https://www.example.com" target="_blank">Visit Example</a>

In this example, href specifies the URL of the link, and target="_blank" makes the link open in a new tab.

HTML Elements

An HTML element is a fundamental building block of HTML documents. Each element consists of an opening tag, content, and a closing tag. The opening tag includes the element name enclosed in angle brackets, while the closing tag has a forward slash before the element name. Elements can also include attributes that provide additional information.

Basic Structure of an HTML Element

Here is a breakdown of the basic structure of an HTML element:


<tagname attribute="value">Content</tagname>

For example:


<p>This is a paragraph.</p>

Common HTML Elements

Below are some commonly used HTML elements:

  • <h1> to <h6>: These tags define headings of various levels. <h1> is the highest level (largest), while <h6> is the lowest level (smallest).
  • <p>: Defines a paragraph. Text enclosed in this tag is displayed with space before and after the paragraph.
  • <a>: Defines a hyperlink. The href attribute specifies the URL of the link. Example: <a href="https://www.example.com">Visit Example</a>
  • <img>: Embeds an image. Requires the src attribute to specify the image source and alt attribute for alternative text. Example: <img src="image.jpg" alt="Description">
  • <ul> and <ol>: Define unordered (bulleted) and ordered (numbered) lists, respectively. List items are defined with <li> tags.
  • <div>: Defines a division or section in a document. It is a block-level container used for grouping and styling purposes.
  • <span>: Defines a section of text or a container for inline elements. It is used for styling purposes and does not create a new block.

Attributes

Attributes provide additional information about an element and are included in the opening tag. They are specified as name-value pairs. For example:


<a href="https://www.example.com" target="_blank" title="Visit Example">Visit Example</a>

In this example, href specifies the destination URL, target="_blank" opens the link in a new tab, and title="Visit Example" provides a tooltip text when the mouse hovers over the link.

Nesting Elements

HTML elements can be nested within one another. Proper nesting ensures that elements are displayed correctly and that the document structure remains valid. For example:


<div>
    <h2>Heading</h2>
    <p>This is a paragraph inside a div.</p>
</div>

HTML Tags

HTML tags are essential components of an HTML document. They are used to create and structure the content of a webpage. Tags are enclosed in angle brackets and usually come in pairs: an opening tag and a closing tag. The opening tag starts an element, while the closing tag ends it.

Basic Structure of HTML Tags

The basic syntax for HTML tags is as follows:


<tagname>Content</tagname>

For example:


<p>This is a paragraph.</p>

Opening and Closing Tags

Most HTML elements require both an opening tag and a closing tag. The opening tag includes the element name in angle brackets, while the closing tag includes a forward slash before the element name. For example:


<h1>This is a heading</h1>

Some tags, known as self-closing tags, do not have a closing tag. They are used for elements that do not have content between opening and closing tags. For example:


<img src="image.jpg" alt="Description" />

Attributes

Tags can include attributes that provide additional information about an element. Attributes are added to the opening tag and are specified in name-value pairs. For example:


<a href="https://www.example.com" target="_blank">Visit Example</a>

In this example, href is an attribute specifying the link destination, and target="_blank" is an attribute specifying that the link should open in a new tab.

Common HTML Tags

Here are some commonly used HTML tags:

  • <html>: The root element of an HTML document.
  • <head>: Contains meta-information about the document, such as the title and links to stylesheets.
  • <body>: Contains the content of the document, such as text, images, and links.
  • <title>: Sets the title of the webpage, which appears in the browser's title bar or tab.
  • <div>: Defines a division or section in the document, used for grouping and styling content.
  • <span>: Defines a section of text or an inline container used for styling purposes.
  • <header>: Represents introductory content or a group of introductory content.
  • <footer>: Represents the footer of a section or page, often containing copyright and contact information.
  • <section>: Represents a standalone section of content within a document.
  • <article>: Represents a self-contained piece of content that could be distributed independently.
  • <aside>: Represents content that is tangentially related to the content around it, such as a sidebar.

HTML Attributes

HTML attributes provide additional information about HTML elements and modify their behavior or appearance. They are specified in the opening tag of an element and typically consist of a name and a value.

Syntax of HTML Attributes

Attributes are included in the opening tag of an element, and are written as name-value pairs. The name and value are separated by an equals sign, and the value is enclosed in double or single quotes. Hereâ€™s the basic syntax:


<tagname attribute="value">Content</tagname>

For example:


<a href="https://www.example.com" target="_blank">Visit Example</a>

In this example, href is an attribute specifying the URL, and target="_blank" is an attribute specifying that the link should open in a new tab.

Common HTML Attributes

Here are some commonly used attributes:

  • href: Specifies the URL for a hyperlink. Used in the <a> tag.
  • src: Specifies the URL of an image. Used in the <img> tag.
  • alt: Provides alternative text for an image if it cannot be displayed. Used in the <img> tag.
  • class: Assigns one or more class names to an element, allowing for CSS styling. Example: class="container"
  • id: Assigns a unique identifier to an element, often used for CSS styling or JavaScript manipulation. Example: id="header"
  • style: Allows inline CSS styling of an element. Example: style="color: red;"
  • target: Specifies where to open a linked document. Example: target="_blank" opens the link in a new tab.
  • title: Provides additional information about an element, often shown as a tooltip when the user hovers over the element.
  • name: Used in form elements to specify the name of the field. Example: name="username"
  • placeholder: Provides a hint or placeholder text within an input field. Example: placeholder="Enter your name"

Case Sensitivity and Quotes

Attribute names are case-insensitive, meaning CLASS and class are treated the same. However, attribute values should be enclosed in quotes (either single or double). Quotes are required if the value contains spaces or special characters. If the value does not contain spaces, quotes are optional but recommended for consistency.

Examples of Attributes in Use

Here are a few examples of how attributes are used within HTML elements:


<img src="logo.png" alt="Company Logo" width="100" height="100">
<a href="https://www.example.com" target="_blank" title="Visit Example">Click Here</a>
<input type="text" name="username" placeholder="Enter your username">

Basic Structure of HTML

Every HTML document follows a standard structure that ensures the document is correctly formatted and displayed by web browsers. This basic structure includes several key components: the document type declaration, and the <html>, <head>, and <body> tags.

Document Type Declaration

The document type declaration, or , is used to specify the version of HTML being used. It is placed at the very beginning of the document and helps browsers render the page correctly. For HTML5, the declaration is:


<!DOCTYPE html>

This declaration tells the browser that the document is an HTML5 document.

<html> Element

The <html> element is the root element of an HTML document. It encloses all the content on the page, except for the declaration.


<html>
    
</html>

<head> Element

The <head> element contains meta-information about the document, such as its title, character set, and links to external resources like stylesheets and scripts. It does not display content directly on the webpage but is essential for defining document properties and including external resources.


<head>
    <meta charset="UTF-8">
    <title>Title of the Document</title>
    <link rel="stylesheet" href="styles.css">
    <script src="script.js"></script>
</head>

<body> Element

The <body> element contains the actual content that will be displayed on the webpage, including text, images, links, and other elements. It is the visible part of the document where all user-facing content is placed.


<body>
    <h1>Welcome to My Website</h1>
    <p>This is a paragraph of text.</p>
    <a href="https://www.example.com">Visit Example</a>
</body>

Complete Example

Hereâ€™s a complete example of a simple HTML document structure:


<!DOCTYPE html>
<html>
    <head>
        <title>My First Webpage</title>
    </head>
    <body>
        <h1>Hello, World!</h1>
        <p>Welcome to my first HTML page.</p>
        <a href="https://www.example.com">Learn more</a>
    </body>
</html>

HTML Headings

Headings in HTML are used to define the title or main sections of a document. They help organize content and provide a hierarchical structure, making it easier for users and search engines to understand the document's layout. HTML offers six levels of headings, from <h1> to <h6>, where <h1> is the highest (most important) level and <h6> is the lowest (least important) level.

Heading Levels

Each heading level is defined using a different tag. Hereâ€™s a breakdown:

  • <h1>: Represents the main heading of the document. There should be only one <h1> per page to maintain semantic structure and SEO best practices.
  • <h2>: Represents subheadings under the <h1> heading. Used for major sections or subsections.
  • <h3>: Represents subheadings under the <h2> heading. Used for further subdivisions of content.
  • <h4>: Represents subheadings under the <h3> heading.
  • <h5>: Represents subheadings under the <h4> heading.
  • <h6>: Represents the lowest level of heading.

Usage Examples

Here are examples of how to use the different heading levels:


<h1>Main Heading</h1>
<h2>Subheading Level 1</h2>
<h3>Subheading Level 2</h3>
<h4>Subheading Level 3</h4>
<h5>Subheading Level 4</h5>
<h6>Subheading Level 5</h6>

These headings help to create a clear content hierarchy. For example:


<h1>Understanding HTML</h1>
<h2>Introduction</h2>
<h3>What is HTML?</h3>
<h2>HTML Elements</h2>
<h3>Basic Elements</h3>
<h4>Headings</h4>
<h4>Paragraphs</h4>

SEO and Accessibility

Proper use of headings improves SEO (Search Engine Optimization) by helping search engines understand the structure and relevance of your content. It also enhances accessibility for users with screen readers by providing a clear document outline. Always use headings in a logical order, from <h1> to <h6>, without skipping levels to maintain a proper hierarchy.

HTML Paragraphs

Paragraphs in HTML are used to group sentences into distinct blocks of text, making content easier to read and organize. The <p> tag defines a paragraph and is one of the fundamental elements for structuring text content on a webpage.

Syntax of Paragraphs

The <p> tag is used to encapsulate a block of text. It automatically adds space above and below the paragraph, separating it from other elements. Hereâ€™s the basic syntax:


<p>This is a paragraph of text.</p>

Example:


<p>HTML (HyperText Markup Language) is the standard language used to create webpages. It structures the content on the web and is essential for building websites.</p>

Multiple Paragraphs

You can use multiple <p> tags to create separate paragraphs, each with its own block of text. The browser automatically inserts margins to space out paragraphs:


<p>This is the first paragraph.</p>
<p>This is the second paragraph, which follows the first one with a space in between.</p>
<p>Additional paragraphs will continue to be separated by space.</p>

Paragraph Formatting

The <p> tag is primarily for text content. To control the appearance of paragraphs, such as alignment, indentation, and spacing, you can use CSS (Cascading Style Sheets). For example:


<style>
    p {
        text-align: justify;
        margin: 20px 0;
        line-height: 1.6;
    }
</style>

In this CSS snippet:

  • text-align: justify; makes the text aligned evenly across the width of the paragraph.
  • margin: 20px 0; adds 20 pixels of space above and below each paragraph.
  • line-height: 1.6; increases the space between lines of text for better readability.

Empty Paragraphs

Using empty <p> tags is not recommended as they create unnecessary spacing in the layout. For creating space, use CSS margin or padding properties instead:


<p></p>

HTML Hyperlinks

Hyperlinks, or links, are essential for navigating between webpages and resources on the web. They are defined using the <a> (anchor) tag. Hyperlinks can point to other webpages, files, email addresses, or locations within the same page.

Basic Syntax

The <a> tag is used to create a hyperlink. The most important attribute of the <a> tag is href, which specifies the URL of the page or resource the link points to. The syntax is as follows:


<a href="URL">Link Text</a>

Example:


<a href="https://www.example.com">Visit Example</a>

In this example, clicking the link text "Visit Example" will navigate to https://www.example.com.

Linking to Different Types of Resources

  • External Links: Links to pages on other websites.
    • 
<a href="https://www.example.com">Visit Example</a>
  • Internal Links: Links to other pages within the same website.
    • 
<a href="about.html">About Us</a>
  • Email Links: Opens the default email client with a new message.
    • 
<a href="mailto:example@example.com">Email Us</a>
  • File Downloads: Links to downloadable files.
    • 
<a href="files/document.pdf">Download PDF</a>
  • Anchor Links: Links to specific locations within the same page. Use the id attribute to define anchor points.
    • 
<a href="#section1">Go to Section 1</a>
<h2 id="section1">Section 1</h2>

Additional Attributes

In addition to href, the <a> tag can include several other attributes:

  • target: Specifies where to open the linked document. For example, target="_blank" opens the link in a new tab or window.
    • 
<a href="https://www.example.com" target="_blank">Visit Example</a>
  • title: Provides additional information about the link, often displayed as a tooltip when the user hovers over the link.
    • 
<a href="https://www.example.com" title="Go to Example">Visit Example</a>
  • rel: Specifies the relationship between the current document and the linked document. For example, rel="noopener noreferrer" improves security when opening links in a new tab.
    • 
<a href="https://www.example.com" rel="noopener noreferrer">Visit Example</a>

Styling Links

CSS can be used to style links, including changing their color, removing underlines, and adding hover effects. For example:


<style>
    a {
        color: blue;
        text-decoration: none;
    }
    a:hover {
        color: red;
        text-decoration: underline;
    }
</style>

In this CSS example:

  • color: blue; sets the link color to blue.
  • text-decoration: none; removes the default underline from links.
  • a:hover { color: red; text-decoration: underline; } changes the link color to red and adds an underline when the user hovers over the link.

HTML Images

Images are an integral part of web content, enhancing visual appeal and providing contextual information. In HTML, images are embedded using the <img> tag. This tag is self-closing and does not require a closing tag.

Basic Syntax

The <img> tag is used to display images. The essential attributes are:

  • src: Specifies the URL of the image file.
  • alt: Provides alternative text that describes the image. This text is displayed if the image cannot be loaded and is used by screen readers for accessibility.

Example:


<img src="image.jpg" alt="A beautiful scenery">

In this example, the image file image.jpg is displayed, and the alternative text "A beautiful scenery" describes the image for users who cannot see it.

Additional Attributes

Besides src and alt, the <img> tag supports several other attributes:

  • width and height: Define the dimensions of the image in pixels.
    • 
<img src="image.jpg" alt="A beautiful scenery" width="600" height="400">
  • title: Provides additional information about the image, often displayed as a tooltip when the user hovers over the image.
    • 
<img src="image.jpg" alt="A beautiful scenery" title="Click to view in full size">
  • loading: Controls when the image is loaded. The loading="lazy" attribute defers loading of the image until it is about to be scrolled into view, improving page load performance.
    • 
<img src="image.jpg" alt="A beautiful scenery" loading="lazy">

Image Formats

Images can be in various formats, such as:

  • JPEG (or JPG): Suitable for photographs and images with many colors.
  • PNG: Ideal for images with transparency and high-quality graphics.
  • GIF: Supports animation and simple graphics with a limited color palette.
  • SVG: Scalable vector graphics, useful for logos and icons as they can be resized without losing quality.

Responsive Images

To make images responsive and adapt to different screen sizes, use the CSS max-width property:


<style>
    img {
        max-width: 100%;
        height: auto;
    }
</style>

In this example, setting max-width: 100% ensures the image scales down to fit the container's width, and height: auto; maintains the image's aspect ratio.

HTML Forms

Forms are a fundamental component of web applications, allowing users to submit data to a server for processing. HTML forms are created using the <form> tag, which acts as a container for various input elements like text fields, radio buttons, checkboxes, and more. Forms are essential for tasks such as user registration, login, and data collection.

Basic Syntax

The <form> tag encloses all the input elements. The most important attributes of the <form> tag are:

  • action: Specifies the URL where the form data should be sent for processing.
  • method: Defines the HTTP method (GET or POST) used to send the form data. GET appends data to the URL, while POST sends data in the request body.

Example:


<form action="submit_form.php" method="post">
    <!-- Form elements go here -->
</form>

Form Elements

Forms can include various input elements:

  • Text Input: Allows users to enter text. Defined using the <input> tag with type text.
    • 
<input type="text" name="username" placeholder="Enter your username">
  • Password Input: Allows users to enter a password. Defined using the <input> tag with type password.
    • 
<input type="password" name="password" placeholder="Enter your password">
  • Textarea: Provides a multi-line text input area. Defined using the <textarea> tag.
    • 
<textarea name="message" rows="4" cols="50">Enter your message here</textarea>
  • Radio Buttons: Allow users to select one option from a set. Defined using the <input> tag with type radio.
    • 
<input type="radio" name="gender" value="male"> Male<br>
<input type="radio" name="gender" value="female"> Female
  • Checkboxes: Allow users to select multiple options. Defined using the <input> tag with type checkbox.
    • 
<input type="checkbox" name="subscribe" value="newsletter"> Subscribe to newsletter
  • Select Menu: Provides a dropdown list of options. Defined using the <select> tag with <option> tags.
    • 
<select name="country">
    <option value="us">United States</option>
    <option value="ca">Canada</option>
</select>
  • Buttons: Define buttons for form submission or other actions. Defined using the <button> tag or <input> tag with type submit or reset.
    • 
<button type="submit">Submit</button>
<button type="reset">Reset</button>

Form Validation

HTML5 introduced built-in form validation, allowing you to enforce rules for input fields. Common attributes include:

  • required: Makes a field mandatory.
    • 
<input type="email" name="email" required>
  • pattern: Specifies a regular expression that the input must match.
    • 
<input type="text" name="phone" pattern="[0-9]{10}" placeholder="Enter a 10-digit phone number">
  • min and max: Define the minimum and maximum values for number inputs.
    • 
<input type="number" name="age" min="18" max="99">
  • maxlength: Limits the number of characters that can be entered.
    • 
<input type="text" name="username" maxlength="15">

Form Submission

When the form is submitted, the data is sent to the server as specified by the action attribute. The method attribute determines how the data is sent:

  • GET: Appends form data to the URL as query parameters. Suitable for simple data retrieval.
  • POST: Sends form data in the request body. More secure and suitable for sensitive or large amounts of data.

HTML Tables

Tables are used to organize and present data in a structured format with rows and columns. HTML tables are created using the <table> tag, with rows defined by the <tr> tag and cells defined by the <td> tag. Tables are useful for displaying tabular data, such as schedules, statistics, and other structured information.

Basic Syntax

The basic structure of an HTML table includes:

  • <table>: The container element for the table.
  • <tr>: Defines a row in the table.
  • <td>: Defines a cell (data) within a row.
  • <th>: Defines a header cell in the table (optional but commonly used).

Example:


<table>
    <tr>
        <th>Header 1</th>
        <th>Header 2</th>
    </tr>
    <tr>
        <td>Data 1</td>
        <td>Data 2</td>
    </tr>
    <tr>
        <td>Data 3</td>
        <td>Data 4</td>
    </tr>
</table>

In this example, the table has two headers and two rows of data.

Table Header

Header cells are defined using the <th> tag. These cells are usually bold and centered by default.


<table>
    <tr>
        <th>Name</th>
        <th>Age</th>
    </tr>
    <tr>
        <td>Alice</td>
        <td>30</td>
    </tr>
    <tr>
        <td>Bob</td>
        <td>25</td>
    </tr>
</table>

Table Attributes

Several attributes can be applied to tables and their elements to enhance their presentation:

  • border: Specifies the width of the border around the table and cells. This is deprecated in HTML5 but can still be used for older browsers.
    • 
<table border="1">
    <!-- Table content -->
</table>
  • cellspacing: Specifies the space between cells (deprecated in HTML5). Use CSS for spacing.
    • 
<table cellspacing="5">
    <!-- Table content -->
</table>
  • cellpadding: Specifies the space between the cell content and its border (deprecated in HTML5). Use CSS for padding.
    • 
<table cellpadding="10">
    <!-- Table content -->
</table>

Table Styles with CSS

Use CSS to style tables for better appearance and readability:


<style>
    table {
        width: 100%;
        border-collapse: collapse;
    }
    th, td {
        border: 1px solid black;
        padding: 8px;
        text-align: left;
    }
    th {
        background-color: #f2f2f2;
    }
</style>

In this CSS example:

  • border-collapse: collapse; ensures that table borders are merged into a single border.
  • border: 1px solid black; adds a solid border around table cells.
  • padding: 8px; adds space inside cells.
  • background-color: #f2f2f2; sets a background color for header cells.

Custom Data Attributes

Custom data attributes enable you to store extra information on HTML elements without using non-standard attributes. These attributes start with data- followed by a name of your choosing.

Syntax

Custom data attributes are written in the form data-*="value", where * can be any string you choose.


<div data-user-id="12345" data-role="admin">
 User Information
</div>

Accessing Data Attributes

You can access and manipulate data attributes using JavaScript:


var element = document.querySelector('div');
var userId = element.getAttribute('data-user-id');
var role = element.dataset.role;
console.log('User ID:', userId); // Outputs: '12345'
console.log('Role:', role); // Outputs: 'admin'

Use Cases

Custom data attributes are useful for:

  • Storing additional information for use in JavaScript.
  • Maintaining a clear separation between data and presentation.
  • Enhancing interactivity and dynamic behavior in web applications.

Form Enhancements with HTML5

HTML5 introduced several new input types and attributes to improve form functionality and user experience. These enhancements include better validation, improved user interfaces, and more specialized input fields.

New Input Types

  • <input type="email">: Ensures the input is a valid email address.
    • 
<form>
 <label for="email">Email:</label>
 <input type="email" id="email" name="email" required>
 <button type="submit">Submit</button>
</form>
  • <input type="tel">: For entering telephone numbers.
    • 
<form>
 <label for="phone">Phone Number:</label>
 <input type="tel" id="phone" name="phone">
 <button type="submit">Submit</button>
</form>
  • <input type="date">: Allows users to select a date from a calendar.
    • 
<form>
 <label for="date">Date:</label>
 <input type="date" id="date" name="date">
 <button type="submit">Submit</button>
</form>
  • <input type="range">: Provides a slider for selecting a value.
    • 
<form>
 <label for="volume">Volume:</label>
 <input type="range" id="volume" name="volume" min="0" max="100">
 <button type="submit">Submit</button>
</form>

Form Attributes

  • placeholder: Provides a hint about what to enter in an input field.
    • 
<input type="text" placeholder="Enter your name">
  • required: Specifies that the input field must be filled out before submitting the form.
    • 
<input type="text" required>
  • pattern: Specifies a regular expression that the input value must match.
    • 
<input type="text" pattern="[A-Za-z]{3,}" title="Three or more letters">
  • autocomplete: Controls whether the browser should automatically complete the input field.
    • 
<input type="text" autocomplete="on">
